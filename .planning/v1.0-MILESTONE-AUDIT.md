### Integration Issue (1 — Low Severity)

**CAT-05 Sidebar Confidence Badge**
`Sidebar.tsx` uses `useCatalogStore.getState()` (non-reactive snapshot) inside a JSX IIFE for the selected-element inspector. If element is selected before async catalog fetch completes, badge won't render and won't re-render when data arrives. The CatalogCard view renders the badge reactively and works correctly. Normal usage (browse catalog first, then configure) is unaffected.

**Fix:** Extract `useCatalogStore(s => s.devices)` at component top level per project memoization pattern.
