  <action>
    **Step 1: Create CommandPalette.tsx**

    Build a new component using shadcn `CommandDialog` (which wraps cmdk + Dialog). The command palette has these groups:

    1. **Navigation** group: Configurator, Catalog Browser, Design Wizard. Each `CommandItem` calls `router.navigate()` to switch routes and closes the palette.

    2. **Devices** group (dynamic, based on search): Use Fuse.js to fuzzy-search the catalog device list. Disable cmdk's built-in filtering with `shouldFilter={false}` on the root `Command` component (per RESEARCH.md Pitfall 7). Pipe the Fuse.js results directly. On select, call `useConfigStore.getState().addElement('device', slug)` and close.

    3. **Connectors** group (dynamic): Same as devices but for connectors. On select, call `addElement('connector', slug)`.

    4. **Export** group: Download JSON, Download DXF, Download OpenSCAD, Download Fusion 360 Script, Copy Share URL. Each calls the corresponding export function (reference ExportTab.tsx for the export logic — import or call the same functions).

    5. **Panel Config** group: Change rack standard (19", 10"), Change U-height (1U-6U). These could use nested subcommands or direct items.

    6. **Actions** group: Undo (Ctrl+Z), Redo (Ctrl+Shift+Z). Call `useConfigStore.getState().undo()` / `.redo()`.

    The Fuse.js search should load device and connector lists from `useCatalogStore.getState()`. Create a single Fuse instance (memoized) that indexes both devices and connectors by name, brand, slug, and category. Apply the search query from `CommandInput` value. When the query is empty, show a curated "recent" or "all commands" view with just the static groups.

    Accept `open` and `onOpenChange` props. The component renders `CommandDialog` which handles its own portal.

    **Step 2: Update useKeyboard.ts**

    Add a module-level or exported state for command palette open/close. Options:
    - A simple Zustand atom (e.g., add `commandPaletteOpen` boolean to a small UI store)
    - Or a React state lifted to __root.tsx and passed via context

    Recommended: Add to the existing keyboard hook. Export a `useCommandPalette` hook that returns `{ open, setOpen }` using React state in the root layout.

    In the keyboard event handler:
    - Detect `Cmd+K` (metaKey + k on Mac) or `Ctrl+K` (ctrlKey + k on non-Mac).
    - When detected, call `setOpen(true)` and `e.preventDefault()`.
    - When `open` is true, early-return from ALL other keyboard shortcut handlers (so Ctrl+Z, Delete, arrow keys, etc. don't fire while the palette is open — cmdk handles its own keyboard events internally).

    **Step 3: Mount in root layout**

    In src/routes/__root.tsx, render `<CommandPalette open={...} onOpenChange={...} />`. Wire the open state to useKeyboard's command palette state.
  </action>
  <verify>

    - `npm run dev` — press Cmd+K (or Ctrl+K): command palette opens with search input and command groups
    - Type a device name (e.g., "USW"): matching devices appear in results
    - Click a device result: element is added to the panel, palette closes
    - Click "Configurator" in Navigation group: navigates to configurator route
    - Press Escape: palette closes
    - While palette is open, pressing Delete or Ctrl+Z does NOT trigger app shortcuts
  </verify>
