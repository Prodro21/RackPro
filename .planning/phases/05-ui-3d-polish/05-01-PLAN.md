    Create `components.json` in project root with:
    ```json
    {
      "$schema": "https://ui.shadcn.com/schema.json",
      "style": "new-york",

      "aliases": {
        "components": "@/components",
        "utils": "@/lib",
        "ui": "@/components/ui",
        "hooks": "@/hooks",
        "lib": "@/lib"
      },
      "iconLibrary": "lucide"
    }
    ```

    **Step 4: Create src/lib/utils.ts** with the shadcn cn() utility:
    ```typescript
    import { type ClassValue, clsx } from "clsx";
    import { twMerge } from "tailwind-merge";

    export function cn(...inputs: ClassValue[]) {
      return twMerge(clsx(inputs));
    }
    ```

    **Step 5: Add shadcn/ui components** via CLI:
    ```bash
    npx shadcn@latest add button input select checkbox slider dialog tooltip card table toggle label sonner command
    ```
    This will generate files into src/components/ui/ (the now-empty directory). If the CLI prompts, answer: overwrite=yes, proceed=yes.

    Replace the existing `@theme` block with the OKLCH-based shadcn variables per RESEARCH.md. Key values:

    - `--primary: oklch(0.72 0.15 185)` (Teal/Cyan accent)
    - `--accent: oklch(0.72 0.15 185)` (Teal/Cyan)
    - `--ring: oklch(0.72 0.15 185)` (Teal/Cyan focus ring)
    - `--radius: 0.375rem`

    Add at the top of the file:
    ```css
    @import "tailwindcss";
    @import "tw-animate-css";

    @custom-variant dark (&:is(.dark *));
    ```

    Keep the existing `:root` variables (--color-bg-primary, etc.) during transition so the app does not break before all components are migrated. Add them alongside the new shadcn vars. Also keep the scrollbar styles and body styles, updating body background to use `var(--background)` instead of `var(--color-bg-primary)`.
