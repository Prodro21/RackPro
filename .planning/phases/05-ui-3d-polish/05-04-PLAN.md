    **Step 1: Download PBR texture maps**

    Download 4 tileable texture maps from CC0 sources (3dtextures.me, freepbr.com, ambientcg.com, or Poly Haven) and save to `public/textures/`. Target resolution: 1024x1024 (good quality, ~100-200KB each as JPEG).

    If free CC0 textures cannot be downloaded programmatically (some sites may block), create procedural textures using Node.js canvas or a script:
    - **brushed-metal-normal.jpg**: Generate horizontal directional grain lines (alternating light/dark blue-purple normal map bands). Use the `sharp` library (already a devDep) to generate a 1024x1024 image with horizontal line patterns representing brushed grain. Normal map convention: R=X, G=Y, B=Z (pointing out = 128,128,255 base).
    - **brushed-metal-roughness.jpg**: Generate with horizontal variation (rougher in some bands, smoother in others) to simulate brush marks.
    - **carbon-fiber-normal.jpg**: Generate a 2x2 twill weave pattern at appropriate scale. Repeat unit of ~4mm.
    - **plastic-layerline-normal.jpg**: Generate horizontal parallel lines at ~0.2mm spacing representing FDM layer lines. Subtle variation.

    Create a script `scripts/generate-textures.ts` that uses `sharp` to procedurally generate these textures if downloading is not feasible. The script should be run once and the output committed to `public/textures/`.

    Alternatively, for a simpler approach: use `THREE.CanvasTexture` at runtime to generate procedural textures in the hook. This avoids asset files entirely but results are less realistic. Given CONTEXT.md specifies "full PBR with texture maps," prefer static assets.

    **Step 2: Create src/hooks/usePanelMaterial.ts**

    ```typescript
    import { useMemo, useRef, useEffect } from 'react';
    import { useTexture } from '@react-three/drei';
    import * as THREE from 'three';

    export type MaterialPreset = 'auto' | 'metal' | 'plastic' | 'carbon';

    export function usePanelMaterial(
      fabMethod: string,
      filamentKey: string,
      override: MaterialPreset = 'auto'
    ): THREE.MeshPhysicalMaterial
    ```

    Implementation:
    1. Load all 4 textures via `useTexture` from drei (provides Suspense integration and caching):
       ```typescript
       const textures = useTexture({
         brushedNormal: '/textures/brushed-metal-normal.jpg',
         brushedRoughness: '/textures/brushed-metal-roughness.jpg',
         carbonNormal: '/textures/carbon-fiber-normal.jpg',
         plasticNormal: '/textures/plastic-layerline-normal.jpg',
       });
       ```
    2. Configure tiling on each texture: `wrapS = wrapT = RepeatWrapping`, `repeat.set(4, 4)` (4x4 tiling across the panel).
    3. Determine the active preset:
       ```typescript
       const preset = override !== 'auto' ? override :
         fabMethod === 'sm' ? 'metal' :
         ['pa-cf', 'pet-cf'].includes(filamentKey) ? 'carbon' : 'plastic';
       ```
    4. Create the material via `useMemo` keyed on `preset`:
       - **metal**: `MeshPhysicalMaterial({ color: '#c0c0c8', metalness: 0.95, roughness: 0.3, normalMap: brushedNormal, roughnessMap: brushedRoughness, clearcoat: 0.3, clearcoatRoughness: 0.2 })`
       - **plastic**: `MeshPhysicalMaterial({ color: '#2a2a35', metalness: 0.0, roughness: 0.85, normalMap: plasticNormal })`
       - **carbon**: `MeshPhysicalMaterial({ color: '#1a1a1a', metalness: 0.1, roughness: 0.4, normalMap: carbonNormal, sheen: 0.5, sheenColor: new Color('#333') })`
    5. Dispose the old material when the preset changes. Use a ref to track the previous material:
       ```typescript
       const prevRef = useRef<THREE.MeshPhysicalMaterial>();
       useEffect(() => {
         return () => { prevRef.current?.dispose(); };
       }, [material]); // cleanup on material change
       prevRef.current = material;
       ```

    Also export wall and ear material variants. For walls and ears:
    - Metal: same as faceplate but slightly darker color
    - Plastic: same material (walls are same color)
    - Carbon: same material

    Or export a `usePanelMaterials` (plural) that returns `{ faceplate, wall, ear, tray }` materials all using the same PBR preset but with appropriate color variations.

    **Step 1: Replace static MeshStandardMaterial constants with usePanelMaterial**

    Remove the existing module-level material constants:
    ```typescript

    const MATERIAL_WALL = new THREE.MeshStandardMaterial({ ... });
    ```

    In the scene component (the one that renders inside `<Canvas>`), call:
    ```typescript
    const fabMethod = useConfigStore(s => s.fabMethod);
    const filamentKey = useConfigStore(s => s.filamentKey);
    const [materialOverride, setMaterialOverride] = useState<MaterialPreset>('auto');
    const materials = usePanelMaterial(fabMethod, filamentKey, materialOverride);
    ```

    Apply the returned materials to the faceplate mesh, wall meshes, and ear meshes. The tray material (MATERIAL_TRAY, MATERIAL_HEX_TRAY) can remain as-is or get a slight PBR upgrade (they are less visible).

    **Step 2: Upgrade Environment**

    Change the `<Environment>` preset from "studio" to "warehouse" for better metallic reflections:
    ```tsx
    <Environment preset="warehouse" />
    ```

    The "warehouse" preset from drei provides a higher-quality HDR with wider light sources that show up well in clearcoat and high-metalness materials.

    **Step 3: Add material override dropdown**

    In the 3D toolbar area (where the wireframe toggle currently lives), add a dropdown/select for material appearance override:
    - Label: A small material icon or text "Material"
    - Options:
      - "Auto (match fab method)" — `'auto'`
      - "Brushed Aluminum" — `'metal'`
      - "Matte Plastic" — `'plastic'`
      - "Carbon Fiber" — `'carbon'`
    - Default: "Auto"
    - On change: update `materialOverride` state.

    This dropdown should use a shadcn Select if Plan 01 has completed (which it will have since Plan 04 depends on Plan 03 which is Wave 1, and Plan 01 is also Wave 1). If shadcn is not available in the 3D Canvas context (it renders outside React DOM), use a regular HTML select positioned as an overlay on top of the Canvas, or render the dropdown outside the Canvas in the toolbar area that wraps it.

    **Step 4: Ensure MATERIAL_TRAY and MATERIAL_HEX_TRAY still work**

    The tray materials can stay as `MeshStandardMaterial` (they are less prominent). Or upgrade them to use a slightly different shade of the active PBR preset. Either approach is fine — do not break tray rendering.

    **Step 5: Verify material disposal**

    When switching between presets (e.g., user toggles fab method from 3dp to sm), the old material must be disposed to prevent GPU memory leaks (per RESEARCH.md Pitfall 5). The usePanelMaterial hook handles this, but verify in practice by switching presets multiple times and checking that GPU memory is stable.
