Create `src/lib/autoLayoutV2.ts` as a pure function with no React or store dependencies. The function signature:

```typescript
export type ConnectorZone = 'between' | 'left' | 'right' | 'split';

export interface LayoutV2Options {
  spacing?: number;           // min gap between elements (mm), default 4
  connectorZone?: ConnectorZone; // where connectors go relative to devices, default 'between'
}

export interface OverflowSuggestion {
  message: string;              // e.g., "Switch to 19\" standard" or "Increase to 2U"
  suggestions: string[];        // actionable suggestions list
  totalWidth: number;           // total width needed (mm)
  availableWidth: number;       // panel width available (mm)
}

export interface LayoutV2Result {
  elements: PanelElement[];
  overflow: OverflowSuggestion | null;
  validationIssues: string[];   // element ids with validation issues
}

export function autoLayoutV2(
  elementDefs: Array<{ type: ElementType; key: string }>,
  panW: number,
  panH: number,
  options?: LayoutV2Options,
): LayoutV2Result
```

Implementation algorithm per CONTEXT.md Section 2 locked decisions:

**Step 1 - Resolve dimensions:** For each elementDef, resolve width/height/name/weight using `lookupDevice()`, `lookupConnector()`, and `FANS` (same pattern as V1's `resolveDims` but also captures weight from DeviceDef.wt or CatalogDevice.weight). Skip unresolvable elements. Separate into devices, connectors, and fans.

**Step 2 - Place devices (weight-aware):** Sort devices by weight descending. Place using alternating ear algorithm: heaviest at left ear edge, next heaviest at right ear edge, alternate inward. Left cursor starts at `spacing` from left edge. Right cursor starts at `panW - spacing` from right edge.

```
for (const dev of sortedByWeight) {
  if (placeLeft) {
    cx = leftCursor + dev.w / 2;
    leftCursor = cx + dev.w / 2 + spacing;
  } else {
    cx = rightCursor - dev.w / 2;
    rightCursor = cx - dev.w / 2 - spacing;
  }
  placeLeft = !placeLeft;
}
```

**Step 3 - Group connectors by type:** Create a `CONNECTOR_FAMILIES` constant map for known connector slug-to-family mappings (per RESEARCH.md Open Question 1):
```typescript
const CONNECTOR_FAMILIES: Record<string, string> = {
  'rj45-keystone': 'rj45', 'rj45-ks': 'rj45',
  'bnc-bulkhead': 'bnc', 'bnc-kfb2': 'bnc',
  'sma-bulkhead': 'sma',
  'fiber-lc': 'fiber', 'fiber-sc': 'fiber',
  'usb-a': 'usb', 'usb-c': 'usb',
  'hdmi-bulkhead': 'hdmi',
  'neutrik-d': 'neutrik-d',
  'iec-c14': 'iec',
};
```
Fall back to slug-prefix extraction for unknown slugs (split on `-`, take first segment if it matches known prefixes `['rj45','bnc','sma','fiber','usb','hdmi']`, otherwise use full slug).

Group connectors by family key. Sort groups alphabetically by family for deterministic output.

**Step 4 - Place connectors in zone:** Based on `connectorZone` option:
- `'between'`: Place connector groups in the gap between placed devices (between leftCursor and rightCursor).
- `'left'`: Place connector groups starting from left edge (before first device). Shift devices rightward if needed.
- `'right'`: Place connector groups starting from right edge (after last device). Shift devices leftward if needed.
- `'split'`: Split connector groups evenly -- first half on left side, second half on right side of devices.

Within each zone, place connectors left-to-right per group (grouped connectors stay adjacent).

**Step 5 - Place fans on rear surface:** Same as V1 -- greedy left-to-right on rear surface.

**Step 6 - Overflow detection:** If any elements don't fit (cursor exceeds panel width), compute overflow suggestions. Suggestions are contextual per CONTEXT.md:
- If current standard is `'10'`, suggest "Switch to 19\" standard (450.85mm panel width)"
- If current uHeight is 1, suggest "Increase to 2U for more vertical space"
- Always show: "Remove items to fit within {panW}mm panel width"

**Step 7 - Post-layout validation:** Check placed elements for overlaps and out-of-bounds using simple AABB intersection checks (do NOT import the full `validateExportConfig` from validation.ts -- that operates on ExportConfig, not PanelElement[]; write lightweight inline checks). Return element IDs with issues in `validationIssues`.

All elements get `surface: 'faceplate'` (except fans which get `'rear'`). Center Y = `panH / 2` for all. Generate unique IDs via `Math.random().toString(36).slice(2, 9)`.

Keep `src/lib/layout.ts` (V1) untouched for backward compatibility with existing MCP tests.
