FIX 1 — Between-zone cursor bounds check (lines 357-362):
In the `placeConnectorsInZone` function's `'between'` case, the cursor advances without checking if the connector exceeds `betweenEnd`. Change the loop to check `cursor + con.w > betweenEnd` before placing. If the connector would exceed betweenEnd, place it AFTER the rightmost device instead (fall through to `devRightEdge + spacing` placement). Specifically:
- Track a `fallbackCursor` starting at `devRightEdge + spacing` (for the right-of-devices overflow area).
- In the loop: if `cursor + con.w / 2 > betweenEnd`, place at `fallbackCursor` instead and advance `fallbackCursor`.
- This ensures connectors that don't fit in the gap spill rightward rather than overlapping devices.

FIX 2 — Zone shift bounds clamping (lines 376-381 left, 397-404 right, 442-464 split):
After ALL zone shift logic completes (for left, right, and split cases), add a post-shift bounds clamping pass on `devicePlacements`. For each device placement:
```
dp.cx = Math.max(dp.el.w / 2, Math.min(panW - dp.el.w / 2, dp.cx));
```
This ensures no device center is positioned such that its edges exceed panel bounds.

Additionally, fix the `split` case (lines 460-462) which currently punts on right-zone overlap with a comment "overflow detection will catch it". Instead, add the same logic as the left case: if `newDevRight > rightConLeft`, shift devices leftward by `(newDevRight - rightConLeft + spacing)` then clamp. But since we're now clamping at the end, the critical fix is the post-shift clamping.

Refactor: Extract the bounds clamping to a helper function `clampDeviceBounds(devicePlacements, panW)` called at the end of EACH case (left, right, split) after the shift logic, and also after the `between` case for safety.

FIX 3 — Overlap-aware overflow detection (lines 474-518):
The current `detectOverflow` only checks if individual element edges exceed `panW`/`panH`. Add a total-width check:
- Sum widths of all faceplate elements (not rear).
- If `totalElementWidth > panW`, trigger overflow even if individual elements are within bounds.
- Also detect pairwise overlaps: reuse the existing `aabbOverlap` function; if ANY pair overlaps, include it in the overflow message.
- Update the overflow message to indicate both edge-overflow and overlap-overflow cases.

Additionally, the `_spacing` parameter is unused. Use it: add spacing between elements when computing total width needed: `totalWidthNeeded = sum(el.w) + spacing * (count - 1)`.

  pushUndo(get());

  const result = autoLayoutV2(elementDefs, panW, panH, options);
