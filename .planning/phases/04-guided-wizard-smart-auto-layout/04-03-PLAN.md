<objective>
Add custom text labels to placed elements with configurable position, auto-numbering for grouped connectors, and category icons. Thread labels through SVG rendering, DXF export (TEXT entities on LABELS layer), OpenSCAD export (debossed text), and design serialization.

Purpose: Labels are first-class fabrication output -- they appear on the physical panel (laser-engraved in sheet metal, debossed in 3D print) and in on-screen rendering, making panels self-documenting without external labeling.
Output: ElementLabel type, label rendering in FrontView, label export in DXF and OpenSCAD, label persistence in serializer.
</objective>

**In `src/components/FrontView.tsx`:**

Add SVG label rendering inside the element rendering loop. For each element with `el.labelConfig?.text`:

1. Compute label position (Y offset from element center):
   - `'above'`: `elementTopY - 6` (6mm above cutout top edge)
   - `'below'`: `elementBottomY + 8` (8mm below cutout bottom edge, default)
   - `'inside'`: `elementCenterY` (centered in cutout)

2. Compute display text with auto-numbering per CONTEXT.md Section 3:
   ```typescript
   function computeAutoNumber(el: PanelElement, allElements: PanelElement[]): number {
     // Find all elements with same type AND same labelConfig.text
     // Sort by X position left-to-right
     // Return 1-based index of this element in that sorted group
     const sameGroup = allElements
       .filter(e => e.type === el.type &&
                    e.labelConfig?.text === el.labelConfig?.text &&
                    e.labelConfig?.autoNumber)
       .sort((a, b) => a.x - b.x);
     return sameGroup.findIndex(e => e.id === el.id) + 1;
   }
   ```
   Continuous numbering by X position regardless of zone (per RESEARCH.md Open Question 3).

3. Stagger collision handling per CONTEXT.md Section 3:
   - Compute all label positions first.
   - For adjacent labels (within 2mm X proximity) that would overlap horizontally, alternate between above and below placement.
   - Use a memoized computation (useMemo keyed on elements array reference) to avoid recalculating on every render frame (per RESEARCH.md Pitfall 2). During drag, show label at configured position; stagger recalculates on drag end (when elements change).

4. Render SVG `<text>` element:
   ```tsx
   <text
     x={cx}
     y={labelY}
     textAnchor="middle"
     dominantBaseline={position === 'inside' ? 'central' : 'auto'}
     fill="#ccc"
     fontSize={5}
     fontFamily="inherit"
     className="pointer-events-none select-none"
   >
     {displayText}
   </text>
   ```

5. Category icon rendering (optional): For elements with `el.labelConfig.icon`, render a small SVG symbol (8x8px) next to the label text. Use simple inline SVG paths for the 4 categories:
   - `network`: ethernet jack icon
   - `video`: camera/monitor icon
   - `audio`: speaker icon
   - `power`: lightning bolt icon

   Keep these as simple path data constants (3-4 path commands each). Position the icon to the left of the label text with 2mm gap.

