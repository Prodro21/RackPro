Undo checkpoint on mount per CONTEXT.md Section 4:
```typescript
const checkpointRef = useRef(false);
useEffect(() => {
  if (!checkpointRef.current) {
    useConfigStore.getState().saveCheckpoint();
    checkpointRef.current = true;
  }
}, []);
```

Cancel wizard handler per CONTEXT.md Section 4 ("same store + undo checkpoint"): save the undo stack depth at mount time, then on cancel, pop undo repeatedly back to that depth. This keeps the undo stack consistent (no orphaned entries) and honors the locked decision to use the undo mechanism rather than a parallel snapshot.

```typescript
const undoDepthRef = useRef<number>(0);
const checkpointRef = useRef(false);
useEffect(() => {
  if (!checkpointRef.current) {
    const store = useConfigStore.getState();
    undoDepthRef.current = store.getUndoDepth();  // exposed from 04-01
    store.saveCheckpoint();
    checkpointRef.current = true;
  }
}, []);

const handleCancel = () => {
  const store = useConfigStore.getState();
  // Pop undo stack back to pre-wizard depth
  while (store.getUndoDepth() > undoDepthRef.current) {
    store.undo();
  }
  sessionStorage.removeItem('rackpro-wizard-step');
  navigate({ to: '/' });
};
```

Navigation guard per CONTEXT.md Section 4 -- use TanStack Router `useBlocker`:
```typescript
const { proceed, reset, status } = useBlocker({
  shouldBlockFn: () => currentStep > 0 && currentStep < STEPS.length - 1,
});
```
When `status === 'blocked'`, show a confirmation dialog with "Leave wizard? You can resume later." options. On confirm, call `proceed()`. On cancel, call `reset()`.

Step rendering: conditionally render the current step component based on `currentStep` index. Pass `onNext` and `onBack` callbacks.

`onNext`: increment step if not at last step. For required steps (0: standard, 1: uHeight), validate that a value has been set before advancing.

`onBack`: decrement step (always allowed per CONTEXT.md -- "freely navigate back to revise earlier steps without losing later choices").

FrontView on the right side: import FrontView and render it. It reads from useConfigStore reactively, so it auto-updates as wizard steps modify the store.

Adding a device: When user clicks "Add to Panel" on a card, call `useConfigStore.getState().addElement('device', slug)`. Then immediately re-run auto-layout:

```typescript
// CRITICAL: Event-driven auto-layout, NOT effect-driven (per RESEARCH.md Pitfall 1)
const handleAddDevice = (slug: string) => {
  const store = useConfigStore.getState();
  store.addElement('device', slug);
  // Re-run auto-layout with all current elements
  const elements = useConfigStore.getState().elements;
  const { panelWidth: panW } = panelDimensions(store.standard);
  const panH = panelHeight(store.uHeight);
  const result = autoLayoutV2(
    elements.map(e => ({ type: e.type, key: e.key })),
    panW, panH,
    { connectorZone: connectorZoneRef.current ?? 'between' }
  );
  store.replaceElements(result.elements);
  if (result.overflow) {
    showToast({ message: result.overflow.message, type: 'warning' });
  }
};
```
