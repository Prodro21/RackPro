FIX 1 — Preview3D.tsx cutout lookup (line 191):
Replace:
```ts
import { CONNECTORS } from '../constants/connectors';
import { DEVICES } from '../constants/devices';
```
With:
```ts
import { lookupDevice } from '../constants/deviceLookup';
import { lookupConnector } from '../constants/connectorLookup';
import { FANS } from '../constants/fans';
```
(FANS import may already exist if fan rendering was added.)

In the `cutoutMeshes` useMemo (line 189-216), change the lookup from:
```ts
const lib = el.type === 'connector' ? CONNECTORS[el.key] : DEVICES[el.key];
```
To a 3-branch lookup that handles device, connector, AND fan:
```ts
let lib: { w: number; h: number; color?: string; cut?: string; r?: number } | undefined;
if (el.type === 'connector') {
  const con = lookupConnector(el.key);
  if (con) lib = con;
} else if (el.type === 'fan') {
  const fan = FANS[el.key];
  if (fan) lib = { w: fan.size, h: fan.size, color: '#555' };
} else {
  const dev = lookupDevice(el.key);
  if (dev) lib = dev;
}
```
This ensures catalog-sourced elements (resolved by lookupDevice/lookupConnector which check the catalog store first, then inline constants, then custom devices) are found.

FIX 2 — Preview3D.tsx retention lips (line 283-284):
The retention lip section filters to `e.type === 'device'` and then checks `if (!DEVICES[el.key]) return null`. Replace with:
```ts
if (!lookupDevice(el.key)) return null;
```
This ensures catalog-sourced devices get retention lips.
