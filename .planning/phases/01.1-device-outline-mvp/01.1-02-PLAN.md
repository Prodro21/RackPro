Create the output directory `public/catalog/outlines/` if it does not exist.

**Priority device list** (compact non-rack-width devices where outline differs most from a rectangle, per RESEARCH.md recommendation):

1. **usw-lite-16-poe** (192x185x43.7mm) -- rounded-rectangle switch, slightly rounded corners
2. **ux7** (117x117x42.5mm) -- rounded cube with curved edges, very non-rectangular
3. **usw-lite-8-poe** (99.6x163.7x31.7mm) -- compact switch
4. **usw-flex** (109x75x28mm) -- small flex switch
5. **usw-flex-mini** (107x70x21.5mm) -- compact flex switch
6. **ucg-ultra** (135.4x135.4x32.25mm) -- square gateway
7. **udm** (110x110x131.7mm) -- cylindrical/unusual shape
8. **unvr** (125.3x125.3x127.4mm) -- cube NVR
9. **rpi5** (85x56x17mm) -- irregular board shape (Raspberry Pi)
10. **atem-mini-pro** (232.2x104.1x35.8mm) -- control surface with irregular profile

For each device, generate **top** and **front** face outlines (side face deferred per RESEARCH.md recommendation):

```bash
# For each device and face:
npx tsx scripts/generate-outline.ts --slug {slug} --face {face} --image {photo_path}
```

**Image sourcing strategy:**
- For Ubiquiti devices: download product photos from techspecs.ui.com or the Ubiquiti store website. Save to a temporary `scripts/photos/` directory (NOT committed to git).
- For non-Ubiquiti (rpi5, atem-mini-pro): source from manufacturer websites.
- Use `curl` or `wget` to download images. If direct download URLs are not accessible, use WebFetch to find image URLs on product pages.
- Alternative: If product photos are unavailable or the API produces poor results for a specific device, hand-craft a simplified SVG path based on the device's known dimensions and shape description. This is acceptable for rectangular devices where the outline is simply a rounded rectangle.

**For devices that are essentially rectangular boxes** (usw-lite-16-poe, usw-lite-8-poe, usw-flex, usw-flex-mini, ucg-ultra, atem-mini-pro): if the Vision API is unavailable or produces poor results, generate a rounded-rectangle path programmatically:
```
M {r} 0 L {W-r} 0 Q {W} 0 {W} {r} L {W} {H-r} Q {W} {H} {W-r} {H} L {r} {H} Q 0 {H} 0 {H-r} L 0 {r} Q 0 0 {r} 0 Z
```
Where r = 2mm (typical corner radius for network equipment). This still provides value over plain rectangles because the exact bounding box is enforced.

**For truly non-rectangular devices** (ux7 with large corner radii, udm cylindrical, rpi5 board): the Vision API or manual path crafting is essential. If API is unavailable, create best-effort outlines based on known dimensions and shape descriptions from product documentation.

After generating each outline, immediately validate it:
```bash
npx tsx scripts/validate-outlines.ts
```

If any outline fails validation, re-run generation with different prompt adjustments or manually correct the path.

**Minimum target:** 5 devices with both top and front outlines passing validation. 10 devices is the stretch goal.

<task type="auto">
  <name>Task 2: Create outline index.json manifest</name>
  <files>
    public/catalog/outlines/index.json
  </files>
  <action>
Create `public/catalog/outlines/index.json` -- a manifest listing all available outlines for app-side lookup.

**Schema:**
```json
{
  "schemaVersion": 1,
  "generated": "2026-02-22T00:00:00Z",
  "outlines": {
    "{slug}": {
      "faces": ["top", "front"],
      "bboxW": {width_mm},
      "bboxD": {depth_mm},
      "bboxH": {height_mm}
    }
  }
}
```

Populate by scanning all SVG files in the outlines directory:
- For each `{slug}-{face}.svg` file, extract slug and face
- Group faces by slug
- Look up bboxW/bboxD/bboxH from `public/catalog/devices.json`

The manifest includes ALL three bbox dimensions regardless of which faces are present, so the app can compute expected viewBox for any face without re-reading the catalog.

This can be generated manually or by adding a `--generate-index` flag to the validate-outlines.ts script. If adding to validate-outlines.ts is cleaner, do that and run it.

Validate the generated index.json:
- Every slug in index.json must exist in devices.json
- Every SVG file in the directory must be referenced in index.json
- bbox dimensions must match catalog dimensions exactly (no tolerance -- these are copied, not measured)
  </action>
  <verify>
`cat public/catalog/outlines/index.json | node -e "const j=JSON.parse(require('fs').readFileSync('/dev/stdin','utf8')); console.log('outlines:', Object.keys(j.outlines).length, 'version:', j.schemaVersion)"` prints outline count >= 5 and schemaVersion 1.
  </verify>
  <done>
index.json exists in public/catalog/outlines/ with schemaVersion 1, lists all generated outlines with correct face availability and bbox dimensions matching catalog.
  </done>
</task>
