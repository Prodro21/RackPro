Create `src/lib/validation.ts` with the following exports:

```typescript
export type ValidationSeverity = 'critical' | 'warning';

export type ValidationCode =
  | 'OPEN_CONTOUR'
  | 'HOLE_TO_EDGE'
  | 'MISSING_DEF'
  | 'OVERLAP'
  | 'OUT_OF_BOUNDS';

export interface ValidationIssue {
  elementId: string;       // element key or id
  elementLabel: string;    // human-readable label
  severity: ValidationSeverity;
  code: ValidationCode;
  message: string;         // descriptive message
  fix?: string;            // actionable fix suggestion with exact values
}

export interface ValidationResult {
  pass: boolean;           // true if no critical issues
  hasCritical: boolean;
  hasWarning: boolean;
  issues: ValidationIssue[];
  summary: {
    critical: number;
    warning: number;
    passed: number;
    total: number;
  };
}

export function validateExportConfig(config: ExportConfig): ValidationResult
```

The `validateExportConfig` function runs these checks IN ORDER on `config.elements`:

1. **Missing definitions check (critical)** — For each element, look up its definition:
   - connector: check `CONNECTORS[key]` AND `useCatalogStore.getState().connectors` map
   - device: check `DEVICES[key]` AND `useCatalogStore.getState().devices` map
   - fan: check `FANS[key]`
   - If lookup returns undefined for all sources, push a critical `MISSING_DEF` issue with fix: `"Remove {label} from the panel or add its definition to the catalog"`
   - Import `lookupConnector` from catalog store and `CONNECTORS`, `DEVICES`, `FANS` from constants.

2. **Closed contour validation (critical)** — Since we generate DXF from code (not parsing files), validate the source geometry. BUT FIRST, upgrade `dxfRect()` in `src/export/dxfGen.ts` to emit LWPOLYLINE instead of 4 individual LINEs:

   **DXF generator fix (prerequisite for meaningful validation):**
   - In `src/export/dxfGen.ts`, replace the `dxfRect()` function body to use `dxfLWPolyline()` instead of 4 `dxfLine()` calls:
     ```typescript
     function dxfRect(x: number, y: number, w: number, h: number, layer: string): string {
       return dxfLWPolyline([
         [x, y], [x + w, y], [x + w, y + h], [x, y + h],
       ], layer);
     }
     ```
   - Similarly, update `dxfTrapezoid()` to use LWPOLYLINE:
     ```typescript
     function dxfTrapezoid(cx: number, cy: number, w: number, h: number, taper: number, layer: string): string {
       const bw = w / 2;
       const tw = w / 2 - taper;
       const hh = h / 2;
       return dxfLWPolyline([
         [cx - bw, cy - hh], [cx + bw, cy - hh],
         [cx + tw, cy + hh], [cx - tw, cy + hh],
       ], layer);
     }
     ```
   - This makes the outer boundary, all rect cutouts, and d-sub cutouts single closed LWPOLYLINE entities, matching the tray generator which already uses LWPOLYLINE correctly.
   - The `dxfRect` function is still called for the outer boundary (line 44) and rect/default cutouts (lines 140-164), so this change affects all rectangular shapes.

   **Validation check (now meaningful):**
   - For `round` and `d-shape` cutouts: always closed by definition (CIRCLE entity in DXF). Skip.
   - For `rect` and `d-sub` cutouts: now emitted as closed LWPOLYLINE (flag=1), guaranteed closed. Skip.
   - For `oblong` cutouts (bore slots): still use individual LINE+arc segments. Flag any oblong with fewer than 4 arc segments as `OPEN_CONTOUR` critical, since insufficient segments create visible gaps. In practice current code uses 12 segments, so this passes — but validates the invariant.
   - This check now provides a genuine guarantee: all closed shapes in the DXF are either CIRCLE, closed LWPOLYLINE, or well-segmented LINE arcs.

3. **Hole-to-edge distance check (warning)** — For each element on the faceplate surface:
   - Import `holeToEdge` from `src/constants/materials.ts`
   - Compute the minimum required distance: `holeToEdge(thickness, radius)` where `thickness` comes from `config.fabrication` (use `thickness` field from FabConfigSM, or `wallThickness` from FabConfig3DP), and `radius` is `el.radius ?? Math.min(el.w, el.h) / 2`.
   - Compute distances from element edges to panel edges: `left = el.x - el.w/2`, `right = panelWidth - (el.x + el.w/2)`, `top = el.y - el.h/2`, `bottom = panelHeight - (el.y + el.h/2)`.
   - If any edge distance < minimum, push a `HOLE_TO_EDGE` warning with exact values: `"Move {label} at least {delta}mm away from the {edge} edge"` where delta = minDist - actualDist.
   - Only run this check for sheet metal fab method (when `config.fabrication.method === 'sm'`). For 3D print, hole-to-edge is less critical.

4. **Element overlap check (critical)** — Iterate all faceplate element pairs. For each pair, check if bounding boxes overlap (AABB intersection). If overlap found, push `OVERLAP` critical issue with fix: `"Move {label1} or {label2} to eliminate overlap"`.

5. **Out-of-bounds check (critical)** — For each element, check if any part extends beyond panel bounds (0 to panelWidth, 0 to panelHeight). If out of bounds, push `OUT_OF_BOUNDS` critical issue with fix: `"Move {label} within panel bounds"`.

Compute summary: count critical, warning, passed (total - critical - warning), total elements. Set `pass = !hasCritical`.

Also re-export from `src/lib/index.ts`:
```typescript
export { validateExportConfig, type ValidationResult, type ValidationIssue, type ValidationSeverity } from './validation';
```
