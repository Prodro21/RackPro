# Plan 08-03: SVG Views + 3D Preview

## Goal
Update FrontView, SideView, SplitView SVG colors for light backgrounds and ensure Preview3D maintains its dark viewport with updated toolbar/overlay styling. After this plan, all visual views look correct against the new light theme.

## Must-Haves

### Truths
- FrontView SVG: dark strokes on light background, orange for selected elements
- SideView SVG: consistent with FrontView color scheme
- SplitView SVG: readable labels, proper contrast
- Preview3D: dark canvas background preserved, toolbar/buttons use light theme
- All hardcoded oklch/hex colors in SVG components updated
- No white-on-white or invisible elements

### Artifacts
- `src/components/FrontView.tsx`
- `src/components/SideView.tsx`
- `src/components/SplitView.tsx`
- `src/components/Preview3D.tsx`

## Key Links
- `08-CONTEXT.md` — Dark 3D viewport decision
- `08-01-PLAN.md` — CSS vars reference

## Execution

### Step 1: FrontView.tsx
- SVG background: white or very light gray
- Panel outline: dark stroke (`oklch(0.3 0 0)`)
- Element fills: light gray default, orange when selected/hovered
- Labels: dark text, readable sizes
- Grid lines: subtle light gray
- Bores/holes: medium gray fill

### Step 2: SideView.tsx
- Match FrontView color scheme
- Cross-section fills: light gray for material, white for voids
- Dimension lines: medium gray with dark text labels
- Orange for highlighted/active dimensions

### Step 3: SplitView.tsx
- Section backgrounds: white
- Split lines: dashed medium gray
- Piece labels: dark text, `text-sm` minimum
- Joint indicators: orange accent
- Remove any tiny text (`text-[7-11px]`)

### Step 4: Preview3D.tsx
- Canvas background: keep dark (studio/warehouse environment)
- Toolbar overlay: light background (`bg-card/90`) with dark text
- Toolbar buttons: standard light theme button styling
- Material labels: readable on dark background (white text with backdrop)
- FAB/action buttons: orange accent

### Step 5: Verify build
Run `npm run build` and fix any breakage.
