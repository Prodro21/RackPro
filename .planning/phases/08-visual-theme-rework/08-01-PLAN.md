# Plan 08-01: Theme Foundation — CSS Variables + Inter Font

## Goal
Replace the dark Slate/Teal theme with a light professional theme using #FF5500 orange accent, Inter font, and updated spacing/radius tokens. After this plan, every component using semantic CSS vars automatically reflects the new palette.

## Must-Haves

### Truths
- All 16 shadcn CSS vars updated to light theme palette
- All 15 legacy `--color-*` vars removed from `:root` and `@theme inline`
- Inter Variable font installed and set as body font
- Monospace retained for `--font-family-mono`
- Base radius updated to 0.5rem (8px)
- Body background is near-white, text is near-black
- Build passes (`npm run build`)

### Artifacts
- `src/index.css` — Complete rewrite of `:root` and `@theme inline` blocks
- `package.json` — `@fontsource-variable/inter` dependency added

## Key Links
- `src/index.css` — Theme definition
- `08-CONTEXT.md` — Locked color values
- `08-RESEARCH.md` — Full palette specification

## Execution

### Step 1: Install Inter font
```bash
npm install @fontsource-variable/inter
```

### Step 2: Rewrite `src/index.css` `:root` block
Replace all CSS variables with light theme palette from RESEARCH.md. Remove all legacy `--color-*` vars.

### Step 3: Update `@theme inline` block
Remove all legacy `--color-*` bridges. Keep only shadcn semantic bridges.

### Step 4: Update body styles
- Font: `"Inter Variable", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- Background: `var(--background)` (now near-white)
- Color: `var(--foreground)` (now near-black)

### Step 5: Add Inter import
Add `@import "@fontsource-variable/inter";` at top of index.css.

### Step 6: Search and replace legacy var usages
Find all `color-accent-gold`, `color-text-primary`, `color-text-secondary`, `color-text-dim`, `color-text-muted`, `color-text-label`, `color-bg-*`, `color-border-dim`, `color-danger` references in components and replace with semantic equivalents.

### Step 7: Verify build
Run `npm run build` and fix any breakage.
