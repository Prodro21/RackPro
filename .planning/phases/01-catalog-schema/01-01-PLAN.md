<task type="auto">
  <name>Task 1: Create Zod v4 catalog schemas</name>
  <files>src/catalog/schemas.ts</files>
  <action>
Create `src/catalog/` directory and `src/catalog/schemas.ts` with the following Zod v4 schemas.

**CRITICAL Zod v4 rules (not v3):**
- `z.record()` requires TWO arguments: `z.record(z.string(), z.string())`
- Use `z.enum()` (not `z.nativeEnum()`)
- `.default()` short-circuits parsing in v4 (returns value without validating it)
- `z.object()` strips unknown keys by default (this is fine for catalog parsing)

**Schemas to define:**

1. `DataSourceSchema` — `z.enum(['manufacturer-datasheet', 'user-calipered', 'cross-referenced', 'estimated'])`

2. `DeviceCategorySchema` — `z.enum(['switch', 'router', 'gateway', 'access-point', 'nas', 'compute', 'converter', 'patch-panel', 'ups', 'pdu', 'other'])`

3. `PortTypeSchema` — `z.enum(['rj45', 'sfp', 'sfp+', 'sfp28', 'qsfp+', 'qsfp28', 'usb-a', 'usb-c', 'console', 'poe-in'])`

4. `PortGroupSchema` — `z.object({ type: PortTypeSchema, count: z.number().int().min(0), speed: z.string().optional(), poe: z.boolean().default(false), pitch: z.number().positive().optional(), positions: z.array(z.number()).optional() })`

5. `CatalogDeviceSchema` — `z.object()` with fields:
   - `slug`: `z.string().regex(/^[a-z0-9-]+$/)` — unique identifier
   - `name`: `z.string().min(1)` — display name
   - `brand`: `z.string().min(1)`
   - `family`: `z.string().optional()` — for Phase 2 UI grouping
   - `category`: `DeviceCategorySchema`
   - `width`, `depth`, `height`: `z.number().positive()` — mm
   - `weight`: `z.number().positive()` — kg
   - `ports`: `z.array(PortGroupSchema).default([])`
   - `color`: `z.string().default('#cccccc')`
   - `portSummary`: `z.string().optional()` — human-readable e.g. "16xGbE (8 PoE+)"
   - `poeBudget`: `z.string().optional()` — e.g. "45W"
   - `power`: `z.string().optional()` — power consumption description
   - `discontinued`: `z.boolean().default(false)`
   - `dataSource`: `DataSourceSchema`
   - `source`: `z.string().optional()` — URL or reference
   - `notes`: `z.string().optional()` — dimension caveats
   - `schemaVersion`: `z.literal(1)`

6. `CutoutTypeSchema` — `z.enum(['round', 'd-shape', 'rect', 'd-sub'])`

7. `ConnectorModuleSchema` — `z.object({ name: z.string(), description: z.string().optional(), partNumbers: z.array(z.string()).optional() })`

8. `CatalogConnectorSchema` — `z.object()` with fields:
   - `slug`: `z.string().regex(/^[a-z0-9-]+$/)`
   - `name`: `z.string().min(1)`
   - `description`: `z.string().optional()`
   - `cutoutType`: `CutoutTypeSchema`
   - `cutoutWidth`: `z.number().positive()` — mm
   - `cutoutHeight`: `z.number().positive()` — mm
   - `cutoutRadius`: `z.number().positive().optional()` — for round/d-shape
   - `mountHoles`: `z.number().int().min(0).default(0)`
   - `mountHoleDiameter`: `z.number().positive().optional()`
   - `mountHoleSpacing`: `z.number().positive().optional()`
   - `depthBehind`: `z.number().positive()` — connector body depth mm
   - `cableClearance`: `z.number().positive().optional()` — minimum cable bend radius mm
   - `color`: `z.string().default('#4a90d9')`
   - `icon`: `z.string().default('*')`
   - `compatibleModules`: `z.array(ConnectorModuleSchema).default([])`
   - `panelThicknessMax`: `z.number().positive().optional()` — e.g. 2mm for keystone snap-in
   - `dataSource`: `DataSourceSchema`
   - `source`: `z.string().optional()`
   - `notes`: `z.string().optional()`
   - `schemaVersion`: `z.literal(1)`

Export ALL schemas and their names. Follow project conventions: named exports only, JSDoc on each schema explaining its purpose.
  </action>
  <verify>Run `npx tsc --noEmit` — no type errors in the new file. Verify file exists and exports all schemas.</verify>
  <done>All Zod v4 schemas compile without errors. CatalogDeviceSchema and CatalogConnectorSchema contain every field specified in CONTEXT.md decisions (4-tier dataSource, port layout, family, discontinued, cutout+module architecture, depthBehind, cableClearance, panelThicknessMax, schemaVersion).</done>

<task type="auto">
  <name>Task 2: Create derived types and backward-compat converters</name>
  <files>src/catalog/types.ts, src/types.ts</files>
  <action>
Create `src/catalog/types.ts` that:

1. Imports schemas from `./schemas`
2. Derives TypeScript types using `z.infer<>`:
   - `export type CatalogDevice = z.infer<typeof CatalogDeviceSchema>`
   - `export type CatalogConnector = z.infer<typeof CatalogConnectorSchema>`
   - `export type DataSource = z.infer<typeof DataSourceSchema>`
   - `export type DeviceCategory = z.infer<typeof DeviceCategorySchema>`
   - `export type PortGroup = z.infer<typeof PortGroupSchema>`
   - `export type ConnectorModule = z.infer<typeof ConnectorModuleSchema>`

3. Creates backward-compat converter functions:

   `export function toDeviceDef(cd: CatalogDevice): DeviceDef` — maps:
   - `name` -> `name`
   - `width` -> `w`
   - `depth` -> `d`
   - `height` -> `h`
   - `weight` -> `wt`
   - `color` -> `color`
   - `portSummary ?? ''` -> `ports`
   - `poeBudget ?? '-'` -> `poe`
   - Compute `portLayout.rj45` and `portLayout.sfp` by summing matching port groups from `cd.ports`

   `export function toConnectorDef(cc: CatalogConnector): ConnectorDef` — maps:
   - `name` -> `name`
   - `description ?? ''` -> `desc`
   - `cutoutType` -> `cut`
   - `cutoutWidth` -> `w`
   - `cutoutHeight` -> `h`
   - `cutoutRadius` -> `r` (only if defined)
   - `color` -> `color`
   - `icon` -> `icon`
   - `depthBehind` -> `depthBehind`
   - `mountHoles` -> `mountHoles`

4. Re-exports all schemas from `./schemas` for convenience.

**Also update `src/types.ts`:** Add `FanDef` interface if it doesn't exist (check first). Do NOT change existing `DeviceDef` or `ConnectorDef` interfaces — the converters bridge the gap.

Follow project naming conventions: PascalCase types, camelCase functions, JSDoc on converter functions.
  </action>
  <verify>Run `npx tsc --noEmit` — no type errors. Verify `toDeviceDef` return type matches `DeviceDef` interface exactly. Verify `toConnectorDef` return type matches `ConnectorDef` interface exactly.</verify>
