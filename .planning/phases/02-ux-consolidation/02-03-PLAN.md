  <action>
    **Fix snap grid dot contrast in `src/index.css`:**
    - Find the `--svg-grid-dot` CSS variable in the dark theme (:root) section
    - Change from `#1e2029` (too close to bg-main `#0f1015`) to `#2a2d3a` (brighter, ~+15 luminance steps)
    - Find the `--svg-grid-dot` in the light theme (.light) section
    - Change from `#d1cfc9` to `#c0bdb5` (slightly darker for better contrast against `#eae9e5`)
    - These values come from the research document's Pitfall 4 analysis

    **Fix grid background visibility:**
    The root cause is that the SVG's full-viewport `<rect>` background fill covers the HTML grid-bg pattern. Two approaches (try in order):

    **Approach A (preferred — SVG-internal grid):**
    In `src/components/FrontView.tsx`, add a secondary SVG `<pattern>` definition alongside the existing snap dot pattern. This pattern renders subtle 40px grid lines inside the SVG viewport:
    ```tsx
    <defs>
      {/* Existing snap dot pattern */}
      <pattern id="g" ...>...</pattern>
      {/* New: background grid lines (40px spacing) */}
      <pattern id="grid-lines" width="40" height="40" patternUnits="userSpaceOnUse">
        <path d="M 40 0 L 0 0 0 40" fill="none" stroke="var(--svg-grid-line, #1a1c24)" strokeWidth="0.5" />
      </pattern>
    </defs>
    ```
    Add a `<rect>` after the canvas background rect that fills with the grid-lines pattern:
    ```tsx
    <rect width="100%" height="100%" fill="url(#grid-lines)" />
    ```
    Add `--svg-grid-line` CSS variable to both themes in index.css:
    - Dark: `--svg-grid-line: #1a1c24` (subtle dark lines on dark bg)
    - Light: `--svg-grid-line: #d8d6d0` (subtle lines on light bg)

    **Approach B (fallback — transparent SVG bg):**
    If Approach A doesn't look right, make the SVG background rect use `fillOpacity={0.92}` instead of opaque fill, so the CSS `.grid-bg::before` pseudo-element grid shows through from behind.

    **Update `src/components/MainContent.tsx`:**
    - Verify the `grid-bg` class is applied to the container div behind the SVG
    - If the grid-bg pseudo-element is no longer needed (because we use SVG-internal grid), consider removing the `.grid-bg::before` styles from index.css — OR keep them as a fallback for non-Front tabs
    - Ensure no z-index stacking context issues prevent the grid from being visible

    **Update `src/hooks/useKeyboard.ts`:**
    - Check if any keyboard shortcuts navigate to removed routes (`/catalog`, `/wizard`)
    - If any exist, replace with modal-opening calls: `useUIStore.getState().openCatalogModal()` etc.
    - If no route-based shortcuts exist, no changes needed — just verify
