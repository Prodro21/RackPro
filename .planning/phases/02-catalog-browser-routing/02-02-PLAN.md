  <action>
Create `src/hooks/useCatalogSearch.ts` implementing Fuse.js fuzzy search composed with category and brand filters.

**Type definitions:**
- Define a union type `CatalogItem` that tags both device and connector entries with an `itemType` discriminator:
  ```typescript
  type CatalogItem =
    | (CatalogDevice & { itemType: 'device' })
    | (CatalogConnector & { itemType: 'connector' });
  ```
- Import `CatalogDevice` and `CatalogConnector` from `../catalog/types`

**Fuse.js configuration (per RESEARCH.md):**
```typescript
const FUSE_OPTIONS: Fuse.IFuseOptions<CatalogItem> = {
  keys: [
    { name: 'name', weight: 0.7 },
    { name: 'brand', weight: 0.5 },
    { name: 'category', weight: 0.3 },
    { name: 'slug', weight: 0.2 },
    { name: 'description', weight: 0.15 },
  ],
  threshold: 0.35,
  distance: 100,
  includeScore: true,
  minMatchCharLength: 2,
  shouldSort: true,
};
```

**Hook implementation:**
1. Read `devices` and `connectors` from `useCatalogStore` using stable selectors (extract at hook top, not inline)
2. Build unified `allItems: CatalogItem[]` via `useMemo` keyed on devices and connectors arrays
3. Create Fuse instance via `useMemo` keyed on `allItems`
4. Manage state: `query` (string), `categories` (Set of strings), `brands` (Set of strings)
5. Filter pipeline in `useMemo`:
   - If query length >= 2: run `fuse.search(query)` and extract `.item` from results
   - Else: use `allItems` (no search filtering)
   - If categories.size > 0: filter to items whose category is in the set (for connectors, check if `'connector'` is in the categories set)
   - If brands.size > 0: filter to items where `itemType === 'device'` AND brand is in the set (connectors have no brand — they pass through if brands filter is active only if the item is not a device, OR filter them out — per CONTEXT.md AND logic means only show items matching ALL active filters, so connectors would be excluded when brand filter is active since they have no brand)
6. Derive `availableCategories` and `availableBrands` from `allItems` via `useMemo` for populating filter controls
7. Return: `{ query, setQuery, categories, toggleCategory, brands, toggleBrand, results, availableCategories, availableBrands, clearFilters }`

**toggleCategory/toggleBrand helper functions:**
- `toggleCategory(cat: string)`: clone Set, add or delete, setCategories
- `toggleBrand(brand: string)`: clone Set, add or delete, setBrands
- `clearFilters()`: reset query to '', categories to empty Set, brands to empty Set

Export as named function: `export function useCatalogSearch()`

CRITICAL: Do NOT subscribe to `useCatalogStore` inline in a callback or conditional. Extract at hook top level per MEMORY.md rules.
  </action>

`npx tsc --noEmit` passes. The hook file has no React hook rule violations (no conditional hook calls).
  </verify>

`useCatalogSearch` hook provides fuzzy search via Fuse.js, category/brand filter composition with AND logic, and derived filter option lists. All selectors are extracted at hook top per MEMORY.md.
  </done>
