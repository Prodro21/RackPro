<task type="auto">
  <name>Task 1: Fix CAT-05 Sidebar reactive selector for confidence badge</name>
  <files>src/components/Sidebar.tsx</files>
  <action>
The Sidebar confidence badge currently uses `useCatalogStore.getState()` inside a JSX IIFE (line ~604-621). This is a non-reactive snapshot — if the user selects an element before the async catalog fetch completes, the badge won't render, and won't re-render when data arrives.

Fix by extracting reactive selectors at the component top level:

1. Near the top of the Sidebar component function (where other hooks are called), add two hook selectors:
   ```ts
   const catalogDevices = useCatalogStore(s => s.devices);
   const catalogConnectors = useCatalogStore(s => s.connectors);
   ```
   NOTE: `useCatalogStore` is already imported at line 5. These selectors return arrays (primitive-stable references from Zustand when the array identity doesn't change), which is fine because catalog data is loaded once and the array reference is stable after load.

2. Replace the IIFE block at line ~604-621:
   ```tsx
   {(() => {
     const catStore = useCatalogStore.getState();
     const entry = selEl.type === 'device'
       ? catStore.devices.find(d => d.slug === selEl.key)
       : selEl.type === 'connector'
       ? catStore.connectors.find(c => c.slug === selEl.key)
       : null;
     if (!entry) return null;
     const badge = confidenceBadge(entry.dataSource);
     return (
       <div className="flex items-center justify-between px-[6px] py-[2px] text-[9px]">
         <span className="text-muted-foreground">Source</span>
         <span className="px-[4px] py-[1px] rounded text-[8px] font-semibold" style={{ background: badge.color + '20', color: badge.color }}>
           {badge.label}
         </span>
       </div>
     );
   })()}
   ```

   With a version that uses the top-level hook variables:
   ```tsx
   {(() => {
     const entry = selEl.type === 'device'
       ? catalogDevices.find(d => d.slug === selEl.key)
       : selEl.type === 'connector'
       ? catalogConnectors.find(c => c.slug === selEl.key)
       : null;
     if (!entry) return null;
     const badge = confidenceBadge(entry.dataSource);
     return (
       <div className="flex items-center justify-between px-[6px] py-[2px] text-[9px]">
         <span className="text-muted-foreground">Source</span>
         <span className="px-[4px] py-[1px] rounded text-[8px] font-semibold" style={{ background: badge.color + '20', color: badge.color }}>
           {badge.label}
         </span>
       </div>
     );
   })()}
   ```

3. Verify that NO other `useCatalogStore.getState()` calls remain in Sidebar.tsx. If any exist, convert them to hook selectors at the top level following the same pattern.

WHY this matters: Per MEMORY.md, React 19 + Zustand 5 requires stable references from selectors. The `.getState()` call returns a snapshot that never triggers re-render. The hook selector `useCatalogStore(s => s.devices)` subscribes to the store and triggers re-render when devices array reference changes (which happens exactly once when async fetch completes).
  </action>

Run `npx tsc --noEmit` — no type errors.
Run `npx vite build` — build succeeds.
  </verify>
  <done>Sidebar confidence badge uses reactive useCatalogStore() hook selectors (not getState() snapshot). Badge will re-render when catalog data arrives even if element was selected before async load completed.</done>
