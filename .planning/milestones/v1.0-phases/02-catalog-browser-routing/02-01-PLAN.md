  <action>
Run `npm install @tanstack/react-router fuse.js` to add both Phase 2 dependencies.

Create `src/router.ts` using code-based routing (NOT file-based routing — no Vite plugin needed):

1. Import `createRootRoute`, `createRoute`, `createRouter`, `createHashHistory`, `lazyRouteComponent` from `@tanstack/react-router`.
2. Create `rootRoute` using `createRootRoute` with its component defined in a separate `__root.tsx` file (use `lazyRouteComponent(() => import('./routes/__root'))`).
3. Create three child routes:
   - `configuratorRoute`: path `'/'`, lazy-loads `./routes/configurator`
   - `catalogRoute`: path `'/catalog'`, lazy-loads `./routes/catalog`
   - `wizardRoute`: path `'/wizard'`, lazy-loads `./routes/wizard`
4. Build route tree: `rootRoute.addChildren([configuratorRoute, catalogRoute, wizardRoute])`
5. Create hash history: `const hashHistory = createHashHistory()`
6. Export router: `export const router = createRouter({ routeTree, history: hashHistory })`
7. Add TypeScript module declaration for type-safe navigation:
   ```typescript
   declare module '@tanstack/react-router' {
     interface Register {
       router: typeof router;
     }
   }
   ```

IMPORTANT: The root route component must NOT be inlined in router.ts. Use lazyRouteComponent to load from `./routes/__root` so the root layout is code-split properly.
  </action>
  <verify>
Run `npx tsc --noEmit` — no type errors. Verify `node_modules/@tanstack/react-router` and `node_modules/fuse.js` exist.
  </verify>
  <done>
`src/router.ts` exports a typed TanStack Router instance with hash history and three route definitions. Both @tanstack/react-router and fuse.js are in package.json dependencies.
  </done>
