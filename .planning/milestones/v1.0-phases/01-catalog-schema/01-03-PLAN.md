Create `src/catalog/useCatalogStore.ts` â€” a Zustand store (separate from useConfigStore per RESEARCH.md recommendation).

**Store interface:**
```typescript
interface CatalogState {
  // Data
  devices: CatalogDevice[];
  connectors: CatalogConnector[];

  // Validation results
  invalidDevices: { index: number; raw: unknown; error: string }[];
  invalidConnectors: { index: number; raw: unknown; error: string }[];

  // Loading state
  loading: boolean;
  ready: boolean;

  // Version tracking for selector memoization
  catalogVersion: string;

  // Actions
  loadCatalog: () => Promise<void>;
}
```

**loadCatalog() implementation:**

1. Set `loading: true`
2. Call `fetchWithCache('/catalog/devices.json')` and `fetchWithCache('/catalog/connectors.json')` in parallel via `Promise.all`
3. For each JSON array, validate **per-entry** using `CatalogDeviceSchema.safeParse()` / `CatalogConnectorSchema.safeParse()`:
   - Valid entries go to `devices[]` / `connectors[]`
   - Invalid entries go to `invalidDevices[]` / `invalidConnectors[]` with the entry index, raw data, and error message string
   - **Per CONTEXT.md decision:** Valid entries load normally regardless of invalid siblings
4. `console.warn` for each invalid entry (developer visibility)
5. Set `loading: false`, `ready: true`, `catalogVersion: \`${devices.length}-${connectors.length}-${Date.now()}\``
6. **Error handling:** If fetch itself fails (network error, 404), `console.error` and set `loading: false` without setting `ready: true`. The app continues working with inline constants as fallback.

**fetchWithCache() helper (private to module):**
```typescript
const CACHE_KEYS = {
  devices: 'rackpro-catalog-devices',
  devicesVer: 'rackpro-catalog-devices-ver',
  connectors: 'rackpro-catalog-connectors',
  connectorsVer: 'rackpro-catalog-connectors-ver',
};

async function fetchWithCache(url: string, cacheKey: string, verKey: string): Promise<unknown[]> {
  const cached = localStorage.getItem(cacheKey);
  const cachedVer = localStorage.getItem(verKey);

  try {
    const res = await fetch(url);
    const etag = res.headers.get('etag') ?? res.headers.get('last-modified') ?? '';

    if (cached && cachedVer && cachedVer === etag && etag !== '') {
      return JSON.parse(cached);
    }

    const data = await res.json();
    try {
      localStorage.setItem(cacheKey, JSON.stringify(data));
      localStorage.setItem(verKey, etag);
    } catch {
      // localStorage quota exceeded â€” proceed without caching
    }
    return data;
  } catch (err) {
    if (cached) return JSON.parse(cached);
    throw err;
  }
}
```

let _devKey = '';

export const selectDeviceMap = (s: CatalogState): Record<string, DeviceDef> => {
  if (s.catalogVersion === _devKey) return _devVal;
  _devKey = s.catalogVersion;

**Update `src/constants/deviceLookup.ts`:**

Replace the existing implementation with a 3-tier lookup:
1. Check catalog store (`useCatalogStore.getState()`) via `selectDeviceMap` â€” this is the primary source after catalog loads
2. Check inline `DEVICES` constant â€” fallback for when catalog hasn't loaded yet or entry not in catalog
3. Check custom devices (`useCustomDevices.getState().customDevices`) â€” user-defined devices

Use module-level cache for the catalog device map to avoid calling `selectDeviceMap` on every lookup:
```typescript
import { useCatalogStore, selectDeviceMap } from '../catalog/useCatalogStore';
import { DEVICES } from './devices';
import { useCustomDevices } from '../store/useCustomDevices';
import type { DeviceDef } from '../types';

export function lookupDevice(key: string): DeviceDef | undefined {
  // 1. Catalog store (fetched JSON â€” primary after load)
  const catMap = selectDeviceMap(useCatalogStore.getState());
  if (catMap[key]) return catMap[key];

  // 2. Inline constants (immediate fallback)
  if (DEVICES[key]) return DEVICES[key];

  // 3. Custom user-defined devices
  const customs = useCustomDevices.getState().customDevices;
  return customs[key];
}
```
