    **Step 1: Create src/lib/csg.ts** — A pure utility module for CSG operations.

    ```typescript
    import { Brush, Evaluator, ADDITION, SUBTRACTION } from 'three-bvh-csg';
    import * as THREE from 'three';
    ```

    Export a `buildCSGFaceplate` function:
    ```typescript
    export interface CutoutDef {
      x: number;      // center X position in mm (panel-local coordinates)
      y: number;      // center Y position in mm
      w: number;      // width for rect cutouts
      h: number;      // height for rect cutouts
      r?: number;     // radius for round/d-shape cutouts
      type: 'rect' | 'round' | 'd-shape';
    }

    export function buildCSGFaceplate(
      panelW: number,
      panelH: number,
      wallT: number,
      cutouts: CutoutDef[],
      scale: number
    ): THREE.BufferGeometry
    ```

    Implementation:
    1. Create the faceplate as a `Brush(new BoxGeometry(panelW*sc, panelH*sc, wallT*sc))`. Call `brush.updateMatrixWorld()`.
    2. If no cutouts, return faceplate geometry directly.
    3. Build a compound cutout brush by iterating cutouts:
       - For `'round'` and `'d-shape'`: Create `CylinderGeometry(r*sc, r*sc, (wallT+0.2)*sc, 32)`. Rotate `Math.PI/2` on X axis so the cylinder axis aligns with Z (through the panel). Per RESEARCH.md Pitfall 8, CylinderGeometry defaults to Y-axis; must rotate to Z.
       - For `'rect'`: Create `BoxGeometry(w*sc, h*sc, (wallT+0.2)*sc)`. The 0.2mm extra depth ensures clean through-cut.
       - Set `brush.position.set((x - panelW/2)*sc, (panelH/2 - y)*sc, 0)` to convert from panel-local coords (origin top-left, Y down) to Three.js centered coords (origin center, Y up). **Critical: Get the coordinate mapping right.** The faceplate mesh is centered at origin. Panel coords: x=0 is left edge, y=0 is top edge. Three.js: x=0 is center, y=0 is center. So: threeX = (x - panelW/2)*sc, threeY = (panelH/2 - y)*sc.
       - Call `brush.updateMatrixWorld()` — CRITICAL per RESEARCH.md Pitfall 3.
    4. Union all cutout brushes into one compound brush using `evaluator.evaluate(compound, brush, ADDITION)`.
    5. Perform a single `evaluator.evaluate(faceBrush, compoundBrush, SUBTRACTION)`.
    6. Return `result.geometry`.

    Also export a cache key helper:
    ```typescript
    export function csgCacheKey(cutouts: CutoutDef[]): string {
      return cutouts.map(c => `${c.type}:${c.x}:${c.y}:${c.w}:${c.h}:${c.r ?? 0}`).join('|');
    }
    ```

    **Step 2: Update Preview3D.tsx — Replace faceplate BoxGeometry with CSG geometry**

    Currently the faceplate is rendered as:
    ```tsx
    <mesh material={MATERIAL_PANEL}>
      <boxGeometry args={[panW*sc, panH*sc, wallT*sc]} />
    </mesh>
    ```

    Replace with a component that:
    1. Collects all placed elements from the store and converts them to `CutoutDef[]`. For each element:
       - Connector: Use `lookupConnector(key)` to get cutout type and dimensions. If cutout is 'round' or 'd-shape', set `r` from connector `r` field. If 'rect', set `w` and `h` from connector `w` and `h`.
       - Device: Set type='rect', w=element.w, h=element.h.
       - Fan: Use FANS[key] for dimensions, typically rect.
    2. Call `buildCSGFaceplate(panW, panH, wallT, cutouts, scale)` inside a `useMemo` keyed on `csgCacheKey(cutouts) + panW + panH + wallT`.
    3. Render: `<mesh geometry={faceplateGeo} material={MATERIAL_PANEL} />`.

    Remove the existing cutout indicator overlays (the dark cylinders/boxes placed ON the faceplate surface). They are replaced by actual holes.

    **Step 3: Error handling**
    Wrap the CSG computation in a try/catch. If CSG fails (e.g., degenerate geometry), fall back to the plain faceplate BoxGeometry and log a warning. The existing error boundary in Preview3D should also catch render-time errors.

    1. Look up the connector definition via `lookupConnector(el.key)`.
    2. Based on the connector type, create the appropriate simplified geometry:
       - **Neutrik D-type** (cutout 'round' or 'd-shape', ~24mm diameter): Cylinder, radius ~12mm, depth = `depthBehind` (~30mm). Color: dark gray.
       - **BNC bulkhead** (cutout 'round', ~9.5mm): Narrow cylinder, depth ~22mm.
       - **Keystone jack** (cutout 'rect', 14.9x19.2mm): Rectangular box, depth ~28mm.
       - **Fiber LC/SC** (cutout 'rect'): Rectangular box with depth from connector spec.
       - **SMA** (cutout 'round', ~6.35mm): Narrow cylinder, depth ~15mm.
       - **D-Sub** (cutout 'rect'/'d-sub'): Rectangular box, slightly tapered if desired but box is fine for low detail.
       - **USB-A, HDMI, IEC** (cutout 'rect'): Rectangular box at specified depth.

    For each:
    - Position: Same X/Y as the cutout, but Z = -(wallT/2 + depthBehind/2) * scale (behind the faceplate, centered on the connector's depth).

    These bodies are LOW-DETAIL — no threads, clips, or pin details. Just spatial awareness shapes.

    For device elements: Do NOT add body geometry. Devices already have trays rendering their spatial footprint.

    For fan elements: Add a simple box at the fan dimensions behind the panel.

    The connector bodies should be rendered as separate meshes (not CSG'd into anything). They sit behind the faceplate and are visible through the CSG cutout holes.
