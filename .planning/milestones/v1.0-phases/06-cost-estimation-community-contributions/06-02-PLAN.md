**`scripts/validate-catalog.ts`:**

Standalone CLI script that validates catalog JSON files. Reuses Zod schemas from `../src/catalog/schemas.ts`.

1. Import `CatalogDeviceSchema` and `CatalogConnectorSchema` from `../src/catalog/schemas.ts` (relative import, works with `npx tsx` per RESEARCH.md)
2. Read `public/catalog/devices.json` and `public/catalog/connectors.json`
3. Parse and validate each entry against Zod schema
4. Check for slug collisions (duplicate slugs within and across files)
5. Check dimension plausibility ranges (per CONTEXT.md):
   - Width: 10-800mm
   - Depth: 10-800mm
   - Height: 10-200mm
   - Weight: 0.01-50kg
6. Output structured results:
   - Write JSON summary to stdout for CI consumption
   - Write human-readable markdown to a temp file (for PR comments)
   - Exit code: 0 if only warnings (plausibility), 1 if schema errors or slug collisions
7. Support `--json` flag for machine-readable output
8. Support `--changed-only <file1> <file2>` flag to only validate specific files (for CI efficiency, but full validation is fine for v1)

Key implementation details:
- Use `fs.readFileSync` for file reading (Node.js script)
- Use Zod `safeParse` for each entry — collect all errors, don't bail on first
- Slug collision: build a Set of all slugs from both files, flag duplicates
- Plausibility: check each numeric dimension against ranges, emit warning (not error)
- Output format for CI: `{ valid: boolean, errors: Array<{slug, field, message}>, warnings: Array<{slug, field, message, value}> }`

**`.github/workflows/validate-catalog.yml`:**

Triggers on PRs touching `public/catalog/**` or `src/catalog/schemas.ts`.

```yaml
name: Validate Catalog PR
on:
  pull_request:
    paths:
      - 'public/catalog/**'
      - 'src/catalog/schemas.ts'

permissions:
  contents: read
  pull-requests: write

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
      - run: npm ci
      - name: Validate catalog entries
        id: validate
        run: npx tsx scripts/validate-catalog.ts --json > validation-result.json 2>&1 || true
      - name: Check result
        id: check
        run: |
          node -e "
            const r = require('./validation-result.json');
            console.log('Valid:', r.valid);
            console.log('Errors:', r.errors?.length ?? 0);
            console.log('Warnings:', r.warnings?.length ?? 0);
            if (!r.valid) process.exit(1);
          "
      - name: Comment PR with results
        if: always()
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            let result;
            try { result = JSON.parse(fs.readFileSync('validation-result.json', 'utf8')); }
            catch { result = { valid: false, errors: [{slug: '', field: '', message: 'Validation script failed to produce output'}], warnings: [] }; }

            let body = '## Catalog Validation Results\n\n';
            if (result.valid) {
              body += '**Status:** All checks passed\n\n';
            } else {
              body += '**Status:** Validation failed\n\n';
            }
            if (result.errors?.length) {
              body += '### Errors (must fix)\n\n| Slug | Field | Message |\n|------|-------|---------|\n';
              for (const e of result.errors) body += `| ${e.slug} | ${e.field} | ${e.message} |\n`;
              body += '\n';
            }
            if (result.warnings?.length) {
              body += '### Warnings (review recommended)\n\n| Slug | Field | Value | Message |\n|------|-------|-------|---------|\n';
              for (const w of result.warnings) body += `| ${w.slug} | ${w.field} | ${w.value} | ${w.message} |\n`;
            }
            if (!result.errors?.length && !result.warnings?.length) {
              body += 'No issues found.\n';
            }

            await github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body
            });
```

**`.github/workflows/auto-generate-entry.yml`:**

Triggers on issues opened with `catalog-submission` label.

```yaml
name: Auto-Generate Catalog Entry
on:
  issues:
    types: [opened, labeled]

permissions:
  contents: write
  pull-requests: write
  issues: write

jobs:
  generate:
    if: contains(github.event.issue.labels.*.name, 'catalog-submission')
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
      - run: npm ci
      - name: Parse issue form
        id: parse
        uses: onmax/issue-form-parser@v1.6.0
        with:
          issue_number: ${{ github.event.issue.number }}
      - name: Generate catalog entry
        id: generate
        run: |
          node -e "
            const payload = JSON.parse(process.env.PAYLOAD);
            // Determine if device or connector from labels
            const isDevice = '${{ contains(github.event.issue.labels.*.name, 'device') }}' === 'true';
            // Build entry object from parsed form fields
            // Map form field IDs to JSON entry fields
            // Write to devices.json or connectors.json
            // Run validation
          "
        env:
          PAYLOAD: ${{ steps.parse.outputs.payload }}
      - name: Validate generated entry
        id: validate
        run: npx tsx scripts/validate-catalog.ts --json > validation-result.json
      - name: Create PR (if valid)
        if: steps.validate.outcome == 'success'
        uses: peter-evans/create-pull-request@v7
        with:
          token: ${{ secrets.GITHUB_TOKEN }}
          commit-message: "feat(catalog): add entry from issue #${{ github.event.issue.number }}"
          branch: "catalog/issue-${{ github.event.issue.number }}"
          title: "feat(catalog): add entry from #${{ github.event.issue.number }}"
          body: |
            Auto-generated from #${{ github.event.issue.number }}

            This PR was automatically created from a catalog submission issue.
            CI will validate the entry. Review and merge if correct.
          labels: catalog-submission
      - name: Comment results on issue
        if: always()
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            let result;
            try { result = JSON.parse(fs.readFileSync('validation-result.json', 'utf8')); }
            catch { result = { valid: false, errors: [{message: 'Generation failed'}], warnings: [] }; }

            let body = '## Auto-Generation Results\n\n';
            if (result.valid) {
              body += 'Entry generated and validated successfully. A draft PR has been created.\n';
            } else {
              body += 'Validation failed. Please check the following errors and update your submission:\n\n';
              for (const e of (result.errors || [])) body += `- **${e.field}**: ${e.message}\n`;
            }

            await github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body
            });
```
