Create `src/catalog/outlines.ts` -- the app-side module for loading and caching SVG outline data.

**Exports:**

```typescript
export type DeviceFace = 'top' | 'front' | 'side';

/** Load the outline index manifest. Called once; cached in module scope. */
export async function loadOutlineIndex(): Promise<OutlineIndex>

/** Get available faces for a device slug (sync, from cached index). Returns empty array if index not loaded or device has no outlines. */
export function getOutlineFaces(slug: string): DeviceFace[]

/** Load an SVG outline path `d` string for a device face. Returns null if no outline exists. Caches in module scope. */
export async function loadOutlinePath(slug: string, face: DeviceFace): Promise<string | null>

/** Check if a device has an outline for a given face (sync, from cached index). */
export function hasOutline(slug: string, face: DeviceFace): boolean

/** Get a cached outline path synchronously (returns null if not yet loaded). For use in synchronous render contexts. */
export function getCachedOutlinePath(slug: string, face: DeviceFace): string | null
```

**Internal state (module-level, NOT in Zustand):**

```typescript
// Module-level cache -- outlines are static assets, not user state.
// Per MEMORY.md pattern: static data uses module-level cache, not Zustand.
let indexPromise: Promise<OutlineIndex> | null = null;
let indexData: OutlineIndex | null = null;
const pathCache = new Map<string, string | null>();
```

**OutlineIndex type:**
```typescript
interface OutlineEntry {
  faces: DeviceFace[];
  bboxW: number;
  bboxD: number;
  bboxH: number;
}

interface OutlineIndex {
  schemaVersion: number;
  outlines: Record<string, OutlineEntry>;
}
```

**loadOutlineIndex implementation:**
- Fetch `/catalog/outlines/index.json`
- On success: parse JSON, store in `indexData`, return it
- On failure (404, network error): set `indexData` to `{ schemaVersion: 1, outlines: {} }` (empty -- no outlines available), log warning
- Deduplicate concurrent calls using the `indexPromise` pattern (if `indexPromise` already exists, return it; otherwise create new fetch promise and store it)

**loadOutlinePath implementation:**
- Check `pathCache` first -- return cached value if present (including cached `null` for missing outlines)
- Ensure index is loaded (await `loadOutlineIndex()`)
- Check if slug+face exists in index. If not, cache `null` and return
- Fetch `/catalog/outlines/{slug}-{face}.svg`
- On success: extract path `d` attribute using regex `d="([^"]+)"`, cache and return
- On failure: cache `null`, return null
- CRITICAL: This is async. Components that need sync access must use `getCachedOutlinePath()` and trigger loading separately.

**getCachedOutlinePath implementation:**
- Return `pathCache.get(key) ?? null` -- purely synchronous, returns whatever is currently cached. Returns null if not yet loaded.

**hasOutline implementation:**
- Return `indexData?.outlines[slug]?.faces.includes(face) ?? false`

**Design decisions:**
- Module-level cache, NOT Zustand store (per RESEARCH.md anti-pattern guidance and MEMORY.md convention for static data)
- Async loading with sync read-back pattern (load triggers fetch, subsequent sync reads hit cache)
- `null` is cached intentionally to avoid re-fetching missing outlines
- No localStorage cache for outlines -- they are small SVGs fetched on demand, and the browser HTTP cache handles repeat visits

import { loadOutlineIndex, loadOutlinePath, getCachedOutlinePath, hasOutline } from '../catalog/outlines';

2. **Add outline loading effect** in the component body. Use a `useState` to track loaded outline keys and trigger re-render:
```typescript
const [outlineVersion, setOutlineVersion] = useState(0);

// Load outline index once on mount
useEffect(() => {
  loadOutlineIndex().then(() => setOutlineVersion(v => v + 1));
}, []);

// Load outlines for placed device elements
useEffect(() => {
  const deviceEls = elements.filter(el => el.type === 'device');
  const toLoad = deviceEls.filter(el => hasOutline(el.key, 'front') && getCachedOutlinePath(el.key, 'front') === null);
  if (toLoad.length === 0) return;
  Promise.all(toLoad.map(el => loadOutlinePath(el.key, 'front'))).then(() => {
    setOutlineVersion(v => v + 1);
  });
}, [elements, outlineVersion]);
```

3. **Replace device rendering** in the elements.map section. For the `el.type !== 'connector' && !isFan` branch (the device branch), replace the rect+X-lines with:

```tsx
{(() => {
  const outlinePath = getCachedOutlinePath(el.key, 'front');
  if (outlinePath) {
    return (
      <path
        d={outlinePath}
        transform={`translate(${ex}, ${ey}) scale(${SC})`}
        fill="#0d0d14"
        stroke={lib && 'color' in lib ? lib.color : '#555'}
        strokeWidth={0.6 / SC}
      />
    );
  }
  return (
    <>
      <rect x={ex} y={ey} width={ew} height={eh}
        rx={1.5} fill="#0d0d14"
        stroke={lib && 'color' in lib ? lib.color : '#555'}
        strokeWidth={0.6} />
      <line x1={ex} y1={ey} x2={ex+ew} y2={ey+eh} stroke="#222" strokeWidth={0.3} />
      <line x1={ex+ew} y1={ey} x2={ex} y2={ey+eh} stroke="#222" strokeWidth={0.3} />
    </>
  );
})()}
```

**Key rendering details:**
- The outline path is in device-local mm coordinates (0,0 to W,H). The `translate` positions it at the element's pixel position, and `scale(SC)` converts mm to SVG pixels (SC is the mm-to-pixel scale factor already used throughout FrontView).
- `strokeWidth` is divided by SC to keep consistent visual stroke width regardless of scale.
- The `fill` and `stroke` colors match the existing rectangle rendering exactly.
- Selection outline, overlap/OOB indicators, margin warnings, and label text all remain UNCHANGED -- they use `ex, ey, ew, eh` which are still computed from `el.w, el.h`. The outline path is just a visual replacement for the rect+X interior.

**IMPORTANT: Do NOT refactor the entire element rendering.** The ONLY change is replacing the 5-line rect+lines block with the conditional path/rect block above. All other rendering (connector circles/rects, fan circles, selection outlines, labels, dimension annotations) stays exactly as-is.
