**Create `src/lib/svgTheme.ts`:**
A centralized SVG color palette object that maps semantic names to CSS variable references. These CSS variables were already defined in index.css (Plan 01) under `--svg-*` namespace.

```typescript
/**
 * Centralized SVG color palette using CSS custom properties.
 * Values are `var(--svg-*)` references that automatically
 * switch between dark and light themes.
 */
export const SVG_COLORS = {
  // Panel body
  panelFace: 'var(--svg-panel-face)',
  panelStroke: 'var(--svg-panel-stroke)',
  panelFaceLight: 'var(--svg-panel-face-light)',   // lighter variant for gradients

  // Mounting ears
  earFill: 'var(--svg-ear-fill)',
  earStroke: 'var(--svg-ear-stroke)',

  // Bores / mounting holes
  boreStroke: 'var(--svg-bore-stroke)',
  boreFill: 'var(--svg-bore-fill)',

  // Canvas
  canvasBg: 'var(--svg-canvas-bg)',
  gridDot: 'var(--svg-grid-dot)',

  // Elements (devices/connectors)
  elementFill: 'var(--svg-element-fill)',
  elementStroke: 'var(--svg-element-stroke)',
  elementText: 'var(--svg-element-text)',
  elementSelected: 'var(--accent)',
  elementSelectedBg: 'var(--accent-subtle)',

  // Dimensions / annotations
  dimLine: 'var(--svg-dim-line)',
  dimText: 'var(--svg-dim-text)',

  // Split lines
  splitLine: 'var(--svg-split-line)',
  splitFill: 'var(--svg-split-fill)',

  // Side view specific
  crossSection: 'var(--svg-cross-section)',
  flangesFill: 'var(--svg-flanges-fill)',
  trayFill: 'var(--svg-tray-fill)',
  wallFill: 'var(--svg-wall-fill)',

  // Text
  textPrimary: 'var(--text-primary)',
  textSecondary: 'var(--text-secondary)',
  textTertiary: 'var(--text-tertiary)',

  // Accent
  accent: 'var(--accent)',
  accentSubtle: 'var(--accent-subtle)',
} as const;
```

**Update `src/components/FrontView.tsx`:**
1. Import `SVG_COLORS` from `src/lib/svgTheme`
2. Systematically replace ALL hardcoded hex colors in SVG elements:
   - Replace fill/stroke attribute hex values with `SVG_COLORS.xxx`
   - For `style={{ fill: '#xxx' }}` patterns, replace with `style={{ fill: SVG_COLORS.xxx }}`
   - For SVG `<rect>`, `<circle>`, `<line>`, `<text>`, `<path>` elements — use the semantic color names from SVG_COLORS

   - Panel strokes → `SVG_COLORS.panelStroke`
   - Ear fills → `SVG_COLORS.earFill`
   - Bore circles → `SVG_COLORS.boreStroke` / `SVG_COLORS.boreFill`
   - Element rectangles → `SVG_COLORS.elementFill` / `SVG_COLORS.elementStroke`
   - Selected elements → `SVG_COLORS.elementSelected`
   - Text labels → `SVG_COLORS.textSecondary` or `SVG_COLORS.elementText`
   - Dimension lines → `SVG_COLORS.dimLine`
   - Background → `SVG_COLORS.canvasBg`
3. Ensure ZERO hardcoded hex colors remain (search for `#[0-9a-fA-F]{3,8}` regex in the file after changes)

Update `src/components/MainContent.tsx` to add the grid background pattern from the mockup:

1. **Background:** `bg-bg-main`

2. **Grid overlay:** Add a CSS pseudo-element or a `::before` overlay div that renders the grid pattern. Use the mockup's approach:
   ```css
   background-image:
     linear-gradient(var(--border-subtle) 1px, transparent 1px),
     linear-gradient(90deg, var(--border-subtle) 1px, transparent 1px);
   background-size: 40px 40px;
   opacity: 0.4;
   ```

   Implementation options (pick the simplest):
   - **Option A (CSS class in index.css):** Add a `.grid-bg` class with the `::before` pseudo-element and apply it to MainContent's container. This is cleanest.
   - **Option B (inline overlay div):** Add an absolutely positioned div inside MainContent with the grid background styles.

   Recommended: Option A — add this to `src/index.css`:
   ```css
   .grid-bg {
     position: relative;
   }
   .grid-bg::before {
     content: '';
     position: absolute;
     inset: 0;
     background-image:
       linear-gradient(var(--border-subtle) 1px, transparent 1px),
       linear-gradient(90deg, var(--border-subtle) 1px, transparent 1px);
     background-size: 40px 40px;
     opacity: 0.4;
     pointer-events: none;
     z-index: 0;
   }
   ```

   Then in MainContent: `className="... grid-bg"` on the outer container. Ensure the content itself has `relative z-10` or similar so it sits above the grid.

3. **Content z-indexing:** The SVG view content and any overlays must render above the grid lines. Set `position: relative; z-index: 1;` on the content wrapper inside MainContent.
